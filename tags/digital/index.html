平时我是挺喜欢用清单类软件来记录的，当你做完一件事，点击完成划勾时，那一抹淡淡的成就感，无与伦比。

我平时用清单软件很常用的一个功能，是做记录类的列表，比如说我有几个清单是：

  1. 待看电影：没看过的方便查，看过的划掉。
  2. 心愿单：平时想剁手的商品太多了，想起来一个就记录下来。
  3. 超市购物单：缺啥了记录下来，比如发现沐浴露快用完了就写上，下次在超市就可以翻看，避免那种买完回家想起来 XX 还没买的窘境。

我现在用的是滴答清单，支持全平台，同步方便。遇到唯一一个困扰就是：对于滴答清单非会员用户，设置一个任务最大只支持19个子任务。大部分情况下对于一个列表是够了，所以也理解官方将其列为会员功能。

但是某些情况比如待看电影，其实经常攒一堆想看的电影放进列表。对于这种偶尔的需求，专程买会员也确实不是刚需，所以我以前都是再建一个列表。

直到刚才我突然想起来，滴答清单最近支持 Markdown 了，而且它的任务详情可以在“列表”和“描述”两种方式切换。

那有没有可能我直接用 Markdown 的任务列表（又叫复选框）语法结合滴答清单的"任务描述"，做无限数量的todolist?

立马测试，我把原本的列表用 Markdown 编辑器 Yu Writer 批量加了"任务列表"格式，复制到滴答清单，结果——

测试证明是可以的！数量不够时不用再建一个列表了，仰天大笑一秒钟！

介绍一下具体用法如下：（估计看这篇的应该不会有人不懂）

- [ ]  待办
- [X] 已完成

上面两行分别表示为未划掉和已划掉的任务。

发现 Windows 自带的“资源监视器”，找出正在上网的程序，功能很有用。不仅可以查看哪个软件占用网速，还可以看到目标主机IP及端口。


默认的启动方式，在任务栏右键点击“任务管理器”，再点“性能”选项卡，再找到左下角的“打开资源监视器”。路径太长了，现在把它加到我的 Quicker 系统面板里。可以随时查看有没有软件偷跑后台下载/上传了。

当然，你喜欢的话，直接放在开始菜单或者桌面都一样。但需要注意快捷方式目标为"%windir%\system32\perfmon.exe /res", 后面需要加一个 /res 的参数，不然会直接打开"性能"监视器，而不是"资源"监视器。
